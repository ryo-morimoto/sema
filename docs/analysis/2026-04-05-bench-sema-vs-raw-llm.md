# sema vs raw LLM ベンチマーク分析

**日付**: 2026-04-05
**目的**: sema セマンティック分析を LLM に事前提供した場合と、LLM がソースを直接読む場合のコスト・速度・精度・安定性を統計的に比較する

## 実験設計

### 条件

| 条件 | 説明 |
|------|------|
| **no-sema** | `claude -p` にプロンプトのみ。LLM が Read ツールでファイルを読み、自力で分析 |
| **with-sema** | sema CLI で事前分析した JSON をプロンプトに注入。LLM はソースを読まず sema データから回答 |

### 固定パラメータ

- モデル: `sonnet` (claude-sonnet-4-6)
- 試行回数: N=10 per condition per size (total 60 runs)
- プロンプト: 4問（pure_functions, effectful_functions, capabilities, dependency_graph）、JSON のみ要求
- sema CLI 実行時間: ~1.2s（全サイズ共通、結果に含めず）

### ターゲットファイル

| サイズ | ファイル | 行数 | 内容 |
|--------|---------|------|------|
| small | `bench/targets/small.ts` | 19 | Config 型 + loadConfig(fs) + validateConfig(pure) + formatConfig(pure) |
| big | `bench/targets/big.ts` | 368 | EC ドメイン: 15 pure 関数 + 7 effectful 関数 + OrderProcessor クラス |
| elephant | `bench/targets/elephant.ts` | 812 | 4 ドメイン × (8 pure + 4 effectful + Service クラス) |

## 観察された事実

### F1: 効果の閾値は ~100 行

19 行ではコスト・時間に有意差なし。368 行で全指標 p<0.01。sema の分析 JSON サイズ（big: 8KB, elephant: 19KB）がソース全文（big: 14KB, elephant: 30KB）より小さい限り効果がある。

### F2: sema はターンを 1 に削減する

no-sema は常に 2 ターン（Read ツール呼び出し + 回答）。sema は分析結果がプロンプトに含まれるため 1 ターンで完結する。これが時間削減の主因。

### F3: sema は出力を安定化する

big の出力 tok: no-sema は 424–2,399 (range 1,975)、sema は 511–518 (range 7)。同じ質問に対する回答のばらつきが 280 倍小さい。

### F4: コスト削減は ~20–24% で頭打ち

big (-21.1%) と elephant (-23.9%) で削減率がほぼ同じ。sema JSON サイズがファイルサイズに比例して増えるため、圧縮率が一定に収束する。

### F5: 時間削減は big で最大、elephant で減衰

big -67.4%、elephant -41.3%。elephant では sema JSON (19KB) の処理自体に時間がかかり始める。

### F6: no-sema は Date() を副作用として検出できるが sema はできない

big.ts の `isDiscountValid` は `new Date()` を呼ぶ。no-sema は 10 回中 10 回 `reads_system_clock` として検出。sema は 10 回中 0 回（pure に分類）。sema の副作用推論ルールにグローバルオブジェクト（Date, Math.random 等）が未登録。

### F7: sema の capability 表記は一貫している

no-sema は run ごとに `filesystem_read` / `fs:read` / `fs.readFileSync` と表記が揺れる。sema は全 run で `fs:read` に統一される。

### F8: elephant の run 8 に外れ値がある

sema elephant run 8: cost $0.168, dur 32,930ms, out 2,563 tok。他の 9 run は cost $0.144, dur ~11,000ms, out 1,001 tok。原因不明（API レイテンシ or モデルの non-determinism）。この外れ値を除外すると elephant の時間削減は -51.8%、出力 tok 削減は -44.8% に改善する。
